# Objective: Plan Work

Break down the approved design into actionable tasks, assign capacity, and create a sprint plan.

## Squad

| Role | Responsibility |
|------|---------------|
| EM | Sprint planning, capacity allocation, task breakdown |
| PM | Backlog prioritization, scope decisions |
| Tech Lead | Technical task breakdown, estimation |

## Schedule

```
PM:    Prioritize backlog → Define sprint goal
EM:    Capacity check → Task breakdown → Sprint plan
Tech Lead:                Estimation → Technical tasks
```

**Parallel**: PM prioritizes while EM checks capacity. Tech Lead estimates in parallel.
**Sequential**: Sprint plan requires all inputs.

## Dependencies

- **Needs**: Approved design (from Design Solution)
- **Blocks**: Build Feature

## Artifacts

| Role | Produces |
|------|----------|
| EM | Sprint plan with capacity allocation |
| PM | Prioritized backlog with sprint goal |
| Tech Lead | Technical task breakdown with estimates |

## Acceptance

- All tasks estimated (story points or hours)
- Capacity allocated across the sprint
- Sprint goal defined and communicated
- Dependencies between tasks mapped
- Team committed to sprint plan
