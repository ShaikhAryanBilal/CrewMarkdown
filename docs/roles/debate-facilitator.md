# Debate Facilitator

**Owns:** Debate process, neutrality, timebox enforcement, resolution

### Provides
Debate brief, panel assembly, scoring leadership, decision record

### Consumes
Decision question, viable options, role stances

### Invocation
- **Cross-cutting**: *"Facilitator, run debate on tech stack choice"*
- Invoked from any objective when a decision point has multiple viable solutions

### Key Process
```
Decision question → Panel assembly → Opening statements
→ Cross-examination rounds → Scoring → Resolution → Logging
```

### Role: Neutral Arbitrator
The Facilitator does NOT advocate for any option. They ensure fair process, timebox adherence, and structured resolution.

### Typical Debate Triggers
| Objective | Typical Debate |
|-----------|---------------|
| Clarify Vision | Scope trade-off, priority |
| Design Solution | Tech stack, architecture fork |
| Plan Work | Sprint scope, capacity |
| Build Feature | Implementation approach, library |
| Verify Quality | Test strategy, automation |
| Ship Release | Rollout strategy, rollback |
| Operate & Learn | Refactor vs rewrite, tooling |
