# Onboarding: Orient

## Need
- Onboarding plan ready
- Newcomer's first day

## Instructions

### 1. Welcome & Big Picture
1. Welcome the newcomer
2. Company/project overview — mission, vision, who we are
3. Team structure — who's who, roles, how teams interact
4. Product overview — what we build, who uses it, why it matters

### 2. Environment Setup
1. Walk through dev environment setup step by step
2. Verify: repo clone, dependencies install, local build runs, tests pass
3. Verify access: all tools, dashboards, communication channels
4. Debug any setup issues together

### 3. First Contribution
1. Find a small, well-scoped first task (documentation fix, small bug, test)
2. Walk through the workflow: branch → code → test → PR → review → merge
3. Buddy reviews the PR and gives constructive feedback
4. First merge — celebrate!

### 4. Knowledge Map
1. Where to find: docs, runbooks, architecture diagrams, decision records
2. Who to ask for: backend, frontend, infra, product, design questions
3. How to escalate: support, incidents, urgent matters
4. Meeting rhythms: standups, planning, retros, demos

## Done
- Newcomer welcomed and understands the big picture
- Dev environment fully set up and verified
- First contribution made and merged
- Knowledge map shared
