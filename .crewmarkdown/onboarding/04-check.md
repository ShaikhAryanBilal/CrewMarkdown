# Onboarding: Check & Adjust

## Need
- Learning sessions underway
- Regular check-ins needed

## Instructions

### 1. Weekly Check-ins (Buddy)
1. What's going well?
2. What's confusing or frustrating?
3. What do you want to learn more about?
4. What wasn't covered that should have been?
5. Rate your confidence (1-5): setup, codebase, domain, process, tools

### 2. Milestone Checks
| Milestone | Check | Success Criteria |
|-----------|-------|-----------------|
| Week 1 | Can they set up and run the project independently? | Yes, with minimal help |
| Week 2 | Have they shipped something to production? | At least one PR merged |
| Week 3 | Can they pick up a small task independently? | Task assigned, understood, in progress |
| Month 1 | Can they own a small feature end-to-end? | From spec to deploy |
| Month 2 | Can they unblock themselves? | Knows where to look/what to ask |

### 3. Adjust the Plan
1. If progressing faster: add more advanced tasks, increase scope
2. If struggling: add more pairing, more guided sessions, more practice tasks
3. If bored: check if tasks are too simple, add variety

### 4. Feedback to Team
1. What gaps in documentation did the newcomer find?
2. What setup friction did they hit?
3. What processes were unclear?
4. Use this to improve onboarding for the next person

## Done
- Regular check-ins conducted
- Milestones assessed and tracked
- Plan adjusted based on progress
- Feedback captured for team improvement
