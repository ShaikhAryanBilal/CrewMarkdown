# Onboarding: Learn & Build

## Need
- Environment set up
- First contribution made
- Basic orientation complete

## Instructions

### 1. Deep Dive Sessions
Schedule guided sessions on:
1. **Architecture walkthrough** — with Architect, review system design, key services, data flow
2. **Codebase tour** — with Tech Lead, explore repo structure, key modules, conventions
3. **Domain deep dive** — with PM/BA, learn domain concepts, user personas, business rules
4. **Tooling overview** — with DevOps, CI/CD pipeline, monitoring, deployment process

### 2. Pairing Sessions
1. Pair with different team members on real tasks
2. Each pairing session has a goal: "By end of this session, you should understand X"
3. Rotate pairs to get diverse perspectives

### 3. Learning Tasks
1. Write a unit test for an existing module
2. Fix a small bug from the backlog
3. Add a small feature with guidance
4. Write/update documentation for something they learned

### 4. Ask Me Anything
1. Regular (weekly) open Q&A sessions
2. No question too small or silly
3. Newcomer can ask about anything: code, process, culture, history

## Done
- Architecture, codebase, domain, and tooling sessions completed
- Multiple pairing sessions with different team members
- Learning tasks completed (test, bug fix, small feature, docs)
- Regular AMA sessions held
