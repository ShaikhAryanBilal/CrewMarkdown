# Review: Prepare

## Need
- Artifact ready for review (code, design, doc, PRD, test plan)
- Review Facilitator or author initiates review

## Instructions

### 1. Identify Artifact
1. What is being reviewed? (link or path)
2. What type: code, architecture, design, PRD, test plan, docs, security
3. What's the review scope? (entire artifact or specific sections)

### 2. Select Reviewers
Use the Review Matrix to pick appropriate reviewers:
| Artifact Type | Suggested Reviewers |
|---------------|-------------------|
| Code | Tech Lead, peer Dev, Security (if auth/crypto) |
| Architecture | Architect, Tech Lead, Security, DevOps |
| Design/UX | UX Designer, PM, Frontend Dev |
| PRD/Requirements | PM, BA, Tech Lead, Architect |
| Test Plan | QA, Dev, PM |
| Documentation | Technical Writer, PM, SME |
| Security | Security, Architect, DevOps |

### 3. Set Review Criteria
1. Define what each reviewer should focus on
2. Set expectations: depth, time commitment, deadline

### 4. Prepare Review Packet
1. The artifact itself
2. Context/background (why this change, what problem it solves)
3. Specific questions or concerns from the author
4. Any relevant docs (tickets, specs, related PRs)

## Done
- Artifact identified with scope
- Reviewers selected
- Criteria defined
- Review packet shared with reviewers
