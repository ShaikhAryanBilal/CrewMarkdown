# Hackathon: Follow-up

## Need
- Hackathon completed
- Projects presented and judged

## Instructions

### 1. Archive Projects
1. Tag all repos: `hackathon-<year>-<event>`
2. Document each project: pitch, team, outcome, demo recording link
3. Store in `.crewmarkdown/logs/<hackathon-date>/hackathon-archive/`

### 2. Capture Learnings
1. What worked well in this hackathon?
2. What could be improved for next time?
3. Any tooling, process, or logistics feedback
4. Write a brief hackathon report

### 3. Track Shippable Projects
1. For projects identified as "most likely to ship":
   - Assign PM to evaluate path to production
   - Create tickets/backlog items
   - Provide resources to continue development
2. Track progress in regular sprint cycles

### 4. Celebrate
1. Share results with broader team/company
2. Highlight key innovations and team achievements
3. Thank mentors, organizers, and participants

## Done
- Projects archived and tagged
- Learnings captured
- Shippable projects tracked
- Event celebrated and shared

→ End of Hackathon workflow
