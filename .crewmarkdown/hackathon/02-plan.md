# Hackathon: Plan & Setup

## Need
- Teams formed with goals
- Hackathon schedule set

## Instructions

### 1. Set Schedule
1. Total hackathon duration (24h, 48h, 1 week, etc.)
2. Key milestones:
   - Start time
   - Checkpoint 1 (progress check)
   - Checkpoint 2 (if long format)
   - Code freeze / demo prep
   - Demo / presentations
   - Judging & awards
   - Wrap-up

### 2. Setup Repos & Environment
1. Each team gets a repo or directory
2. Set up any shared infrastructure
3. Provide access to APIs, keys, datasets
4. Ensure dev environment is ready

### 3. Assign Mentors
1. Identify mentors for each team (if available)
2. Mentors rotate between teams
3. Mentors help with technical blockers, not build the solution

### 4. Define Judging Criteria
| Criterion | Weight | Description |
|-----------|--------|-------------|
| Innovation | 25% | Novelty of the idea |
| Impact | 25% | Potential value of the solution |
| Execution | 25% | How well was it built? |
| Presentation | 15% | Demo quality, clarity |
| Teamwork | 10% | Collaboration, use of skills |

## Done
- Schedule set with milestones
- Repos and environments ready
- Mentors assigned (if available)
- Judging criteria defined and shared
