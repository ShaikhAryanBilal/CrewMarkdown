# Hackathon: Build

## Need
- Teams formed, goals defined, environment ready

## Instructions

### 1. Build Phase
1. Teams work on their projects
2. Follow the schedule: build → checkpoint → build → code freeze
3. During checkpoints: each team shares progress (2 min), blockers identified, mentors assigned to unblock

### 2. Hackathon Champion Checks
1. Circulate between teams
2. Check: are they on track? blocked? need anything?
3. Track progress against milestones
4. Handle any logistical issues

### 3. Code Freeze
1. At code freeze time, all development stops
2. Teams prepare their demo (slides, script, environment)
3. No more code changes after freeze

### 4. Final Prep
1. Test demo one time before presenting
2. Have a backup plan if demo fails (screenshots, video)
3. Prepare for Q&A

## Done
- Build completed within timebox
- Code frozen
- Demo prepared
- Ready to present
