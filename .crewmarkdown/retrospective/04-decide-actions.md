# Retrospective: Decide Actions

## Need
- Root causes identified
- Insights captured

## Instructions

### 1. Brainstorm Actions
1. For each root cause/theme, what can we DO about it?
2. Encourage concrete, specific actions (not vague "improve communication")
3. Focus on actions within the team's control

### 2. Use SMART Criteria
Each action should be:
- **S**pecific — what exactly will be done?
- **M**easurable — how will we know it's done?
- **A**chievable — can we actually do this in the next sprint?
- **R**elevant — does it address the root cause?
- **T**ime-bound — when will it be done?

### 3. Assign Owners & Deadlines
1. Each action item needs: owner, deliverable, deadline
2. No more than 2-3 action items per retro (focus on what you'll actually do)

### 4. Choose Improvement Experiments
1. For the top 1-2 actions, commit to trying them next sprint
2. These become sprint-level improvement experiments
3. Review at next retro: did it work? should we continue, adjust, or drop?

## Done
- Actions defined using SMART criteria
- Owners and deadlines assigned
- Top 1-2 actions committed as experiments
- Ready to log
